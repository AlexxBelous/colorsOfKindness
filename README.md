# 🎨 WP-Docker: NovaTheme Development Environment
Среда разработки для WordPress на базе Docker.

## 🚀 Быстрый старт

### 1. Получение шаблона проекта
Склонируйте репозиторий с базовой конфигурацией:

```bash
git clone git@github.com:AlexxBelous/wp-docker.git
````

### 2. Настройка переменных окружения

Скопируйте пример конфига и при необходимости измените порты или пароли:

```bash
cp .env.example .env
```

### 3. Запуск контейнеров

```bash
docker compose up -d
```

После запуска сайт будет доступен по адресу: [http://localhost:8000](http://localhost:8000)
Панель управления базой данных (phpMyAdmin): [http://localhost:8181](http://localhost:8181)

## 🗂 Структура проекта

```
wp-docker/
├── themes/NovaTheme/      # Исходный код темы (отслеживается Git)
├── plugins/               # Плагины (игнорируются Git)
├── .srv/
│   └── custom.ini         # Настройки PHP
├── .env                   # Конфигурация (не коммитится)
├── .env.example           # Шаблон конфигурации
└── docker-compose.yml     # Docker конфигурация
```

## 🎨 Разработка темы (NovaTheme)

В теме используется Vite и PostCSS.

```bash
# Перейдите в папку темы
cd themes/NovaTheme

# Установите зависимости
npm install

# Запустите режим разработки
npm run dev

# Соберите билд для продакшена
npm run build
```

## 🐳 Управление Docker на Ubuntu
| Команда                           | Описание                                                                                        |
| --------------------------------- | ----------------------------------------------------------------------------------------------- |
| `sudo systemctl start docker`     | Запустить службу Docker на Ubuntu. После этого можно использовать команды Docker.               |
| `sudo systemctl stop docker`      | Остановить службу Docker. Все контейнеры при этом продолжат существовать, но работать не будут. |
| `sudo systemctl restart docker`   | Перезапустить службу Docker. Полезно при сбоях или после обновления конфигурации.               |
| `sudo systemctl status docker`    | Проверить статус службы Docker, увидеть, работает ли она, время запуска, ошибки.                |
| `sudo systemctl enable docker`    | Включить автозапуск Docker при старте системы.                                                  |
| `sudo systemctl disable docker`   | Отключить автозапуск Docker при старте системы.                                                 |
| `docker info`                     | Подробная информация о текущей установке Docker: версии, контейнеры, образы, тома, сети.        |
| `docker version`                  | Показывает версии Docker Engine и клиента.                                                      |
| `docker ps`                       | Список **только запущенных контейнеров** на хосте (все проекты и контейнеры).                   |
| `docker ps -a`                    | Список **всех контейнеров**, включая остановленные, на хосте.                                   |
| `docker logs <container_name>`    | Просмотр логов конкретного контейнера.                                                          |
| `docker stats`                    | Мониторинг использования ресурсов (CPU, память) запущенных контейнеров в реальном времени.      |
| `docker top <container_name>`     | Список процессов внутри контейнера, как `ps` в Linux.                                           |
| `docker inspect <container_name>` | Подробная информация о контейнере или образе: конфигурация, сети, монтирования томов.           |


## 🛠 Полезные команды Docker Compose и очистки
| Команда                                                              | Описание                                                                                                                                                    |
| -------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `docker compose up -d`                                               | Запуск проекта из `docker-compose.yml` в фоне. Контейнеры будут работать в фоне.                                                                            |
| `docker compose ps`                                                  | Список контейнеров **только текущего проекта Compose**, с их статусом (Up, Exited и т.д.).                                                                  |
| `docker compose stop`                                                | Остановить контейнеры проекта Compose без удаления.                                                                                                         |
| `docker compose down`                                                | Остановить контейнеры проекта и удалить их вместе с сетью и томами по необходимости.                                                                        |
| `docker compose logs -f`                                             | Просмотр логов контейнеров проекта в реальном времени (подходит для отладки).                                                                               |
| `docker exec -it <container_name> bash`                              | Зайти внутрь контейнера с Bash (если он установлен).                                                                                                        |
| `docker exec -it <container_name> sh`                                | Зайти внутрь контейнера с sh (для минимальных образов типа Alpine).                                                                                         |
| `docker stop $(docker ps -q)`                                        | Остановить все запущенные контейнеры на хосте.                                                                                                              |
| `docker rm $(docker ps -a -q)`                                       | Удалить все контейнеры на хосте (остановленные и работающие, если они сначала остановлены).                                                                 |
| `docker rmi $(docker images -q)`                                     | Удалить все образы Docker на хосте.                                                                                                                         |
| `docker volume prune -f`                                             | Удалить все неиспользуемые тома.                                                                                                                            |
| `docker network prune -f`                                            | Удалить все неиспользуемые сети.                                                                                                                            |
| `docker system prune -a -f --volumes`                                | Полная очистка Docker: останавливает все остановленные контейнеры, удаляет неиспользуемые образы, тома и сети. **Аккуратно — данные томов тоже удаляются.** |
| `docker stop $(docker ps -q) && docker system prune -a -f --volumes` | Полный сброс Docker в одной команде: сначала останавливает все контейнеры, потом полностью очищает систему.                                                 |
